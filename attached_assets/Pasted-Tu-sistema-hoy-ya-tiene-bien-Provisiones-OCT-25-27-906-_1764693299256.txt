1. Modelo objetivo para CashFlow
1.1. Tabla cash_movements

La tabla cash_movements debe representar cada movimiento de caja de la pestaña CashFlow.

Campos mínimos a usar:

id (ya existe)

date → Fecha del movimiento

period_key → "YYYY-MM" calculado desde Fecha

bank → columna Banco

currency → Moneda

type → "IN" / "OUT"

amount_usd → monto del movimiento en USD (positivo)

concept → Concepto (o Detalle Operación si está más completo)

Reglas:

Filtrar solo filas donde Ingreso/Egreso ∈ {"Ingreso", "Egreso"}

Ignorar filas con Ingreso/Egreso = "Saldo" (son estados, no movimientos).

2. ETL de pestaña CashFlow → cash_movements
2.1. Lógica de parsing

Pseudo-lógica:

Leer la hoja CashFlow desde googleSheetsWorking.ts (o el servicio equivalente).

Para cada fila:

Si Ingreso/Egreso no es "Ingreso" ni "Egreso" → skip.

date = valor de columna Fecha.

period_key = ${date.getFullYear()}-${(date.getMonth()+1).toString().padStart(2, "0")}

bank = Banco

currency = Moneda

concept = preferentemente Concepto (fallback Detalle Operación si viene vacío)

Cálculo de amount_usd:

Si Monto USD tiene valor numérico:

raw = Monto USD

Si Monto USD está vacío y Monto ARS + Cotización tienen valor:

raw = Monto ARS / Cotización

Si no hay forma de calcular → loguear fila como rechazada y seguir.

Normalización por tipo:

Si Ingreso/Egreso = "Ingreso":

type = "IN"

amount_usd = Math.abs(raw)

Si Ingreso/Egreso = "Egreso":

type = "OUT"

amount_usd = Math.abs(raw)

Insert/Upsert en cash_movements.

2.2. Idempotencia

Antes de insertar movimientos de un period_key dado, hacer:

DELETE FROM cash_movements WHERE period_key = $period_key

Luego insertar todos los movimientos de ese mes.

El ETL general puede:

borrar por rango (BETWEEN 'YYYY-01' AND 'YYYY-12') o

borrar todo y reinsertar (si el rendimiento es aceptable y no necesitás histórico manual).
