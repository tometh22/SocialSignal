Ese error viene de un spread sobre algo undefined/null cuando cambiás el período:
[plugin:runtime-error-plugin] Spread syntax requires ...iterable not be null or undefined
Eso pasa con:
...[obj] si obj es undefined/null (en objetos también rompe: {...undefined}),
[...arr] si arr es undefined/null.
Te dejo un plan corto y quirúrgico para que no vuelva a pasar, más los snippets exactos.
