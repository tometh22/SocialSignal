A. Derivar “activo” cuando no venga status

Pegá esto en tu archivo (usamos exactamente tus ProjectItem):

function hasActivityThisPeriod(p: ProjectItem) {
  const m = p.metrics || {};
  return !!(
    (m.revenueDisplay && m.revenueDisplay !== 0) ||
    (m.costDisplay && m.costDisplay !== 0) ||
    (m.revenueUSDNormalized && m.revenueUSDNormalized !== 0) ||
    (m.costUSDNormalized && m.costUSDNormalized !== 0)
  );
}

function isActiveForPeriod(p: ProjectItem, period: string) {
  // 1) si viene status explícito, respetar
  if (typeof p.status === 'string') return p.status !== 'Inactive';

  // 2) si el backend empieza a mandar rango de vigencia (opcional)
  // @ts-ignore tolerante: solo usamos si llega
  const start = (p as any).startMonthKey as string | undefined;
  // @ts-ignore
  const end = (p as any).endMonthKey as string | undefined;
  // @ts-ignore
  const tags = (p.tags || []).map(t => t.toLowerCase());

  const isPuntual = tags.includes('one-shot') || tags.includes('puntual');

  if (isPuntual && (start || end)) {
    const geStart = !start || period >= start;
    const leEnd   = !end   || period <= end;
    return geStart && leEnd;
  }

  // 3) fallback universal: activo si hubo movimiento en el mes
  return hasActivityThisPeriod(p);
}

Datos para “Acumulado/Total”: no rompemos si el endpoint no existe → ocultamos el botón o mostramos tooltip.
Hook simple (usa React Query; si 404/501, devolvemos null y mantenemos “Mes”):

type RollupScope = 'acum'|'total';
type RollupResp = {
  revenueDisplay?: number;
  costDisplay?: number;
  revenueUSDNormalized?: number;
  costUSDNormalized?: number;
  markup?: number;
  margin?: number;
};

function useProjectRollup(projectKey?: string, period?: string, scope?: RollupScope) {
  const enabled = !!projectKey && !!period && !!scope;
  return useQuery({
    queryKey: ['project-rollup', projectKey, period, scope],
    enabled,
    queryFn: async () => {
      const url = new URL(`/api/projects/${encodeURIComponent(projectKey!)}/rollup`, window.location.origin);
      url.searchParams.set('scope', scope!); // 'acum' | 'total'
      url.searchParams.set('thru', period!);
      const r = await fetch(url.toString());
      if (!r.ok) return null; // fallback silencioso si no existe
      return (await r.json()) as RollupResp;
    },
    staleTime: 60_000
  });
}

C. “Marcar como terminado”

Acción opcional que solo aparece si tenemos projectKey:

async function markFinished(projectKey: string, period: string) {
  try {
    const r = await fetch(`/api/projects/${encodeURIComponent(projectKey)}/status`, {
      method: 'PATCH',
      headers: {'Content-Type':'application/json'},
      body: JSON.stringify({ status: 'Inactive', endMonthKey: period })
    });
    if (!r.ok) throw new Error(await r.text());
    // ideal: invalidar cache
  } catch(e) {
    alert('No se pudo marcar como terminado. Ver detalle en consola.');
    console.error(e);
  }
}

Backend (mínimo para soportar lo nuevo)

Nuevo endpoint opcional:
GET /api/projects/:key/rollup?scope=acum|total&thru=YYYY-MM

acum: suma ingresos/costos hasta thru (incl.).

total: vida completa del proyecto (si ya terminó).

Agregar metadatos (opcionales) en /api/projects

startMonthKey, endMonthKey, isFinished, supportsRollup

projectType: 'Fee' | 'Puntual'

Nada es obligatorio: el front cae en gracia si no llegan.

SQL orientativo para rollup acum:

SELECT
  SUM(revenue_usd_normalized)  AS revenueUSDNormalized,
  SUM(cost_usd_normalized)     AS costUSDNormalized
FROM project_ledger
WHERE project_key = $1
  AND month_key <= $2;
