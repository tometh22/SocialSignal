Cambios concretos (para que funcione con todos los proyectos)

Data source (SOLO costos)

Leer de direct_costs / “Costos directos e indirectos”.

Columnas a mapear exactamente:

cliente → clientName

proyecto → projectName

mes (ej. “08 ago”) + año → periodKey (“2025-08”)

costoTotal (ARS), montoTotalUSD (USD)

tipoGasto → filtrar “Directo/Directos”

Eliminar cualquier regla anti-×100/×1000 en costos.

Regla de moneda & normalización (por fila)

Si montoTotalUSD > 0 →
costDisplay = {amount: montoTotalUSD, currency: "USD"} y costUSDNormalized = montoTotalUSD.

Si solo hay costoTotal (ARS) →
costDisplay = {amount: costoTotal, currency: "ARS"} y costUSDNormalized = costoTotal / fx(periodKey).

FX de costos (agosto 2025) = 1.345. Asegurar que costos usen este FX (no el 1.260 de ingresos).

Filtro temporal robusto

Construir periodKey = YYYY-MM desde mes (ES) + año (soporta “08 ago”, “05 may”…).

Filtrar exclusivamente por periodKey.

Agregación

Agrupar por clientName + projectName + periodKey.

Sumar por proyecto: sum(costDisplay.nativo) y sum(costUSDNormalized).

Endpoints SoT de costos (ya listos o ajustar)

GET /api/costs?period=YYYY-MM → lista por proyecto (incluye costDisplay, costUSDNormalized).

GET /api/projects/:client/:project/costs?period=YYYY-MM → detalle.

GET /api/portfolio/costs?period=YYYY-MM → totales del período.

Active Projects / Proyecto Único / Dashboard deben consumir estos endpoints.
