Resumamos y cerremos esto con un plan corto y ejecutable para que los COSTOS (agosto 2025) salgan exactos y confiables en todas las vistas.

1) Causas raíz (confirmadas)

USD corrupto en BD (monto_total_usd astronómico).

month_key vs mes/año: filas con mes="08 ago" pero month_key="2025-01" → caían en agosto por parsing.

FX de costos: debía ser 1345 (no 1260).

Cache: devolviendo resultados viejos tras arreglos.

2) Parches mínimos (código)

Aplica estos 3 cambios y recompila.

(A) Periodo: usar month_key como verdad
// costs/parser.ts
const periodKey =
  isNonEmpty(row.month_key) ? String(row.month_key) :
  buildFromMesAnio(row.mes, row["año"]);  // fallback sólo si falta month_key
if (!/^\d{4}-\d{2}$/.test(periodKey)) skip("bad periodKey");

(B) Moneda & anticorrupción USD (costos)
// costs/business-rules.ts
const fx = getFxCosts(periodKey); // 1345 en 2025-08

const usdRaw = asNumber(row.monto_total_usd);
const arsRaw = asNumber(row.costo_total);

const usdFromArs = isFinite(arsRaw) ? arsRaw / fx : null;

// 1) Si USD es astronómico o > 100k, IGNORAR USD y usar ARS/fx
const usdLooksCorrupt =
  !isFinite(usdRaw) ||
  usdRaw > 100_000 ||
  (usdFromArs && isFinite(usdRaw) && Math.abs(usdRaw - usdFromArs) / Math.max(usdRaw, usdFromArs) > 0.35);

if (!usdLooksCorrupt && isFinite(usdRaw) && usdRaw > 0) {
  costDisplay = { amount: usdRaw, currency: "USD" };
  costUSDNormalized = usdRaw;
} else if (isFinite(arsRaw) && arsRaw > 0) {
  costDisplay = { amount: arsRaw, currency: "ARS" };
  costUSDNormalized = arsRaw / fx;
} else {
  skip("no valid amounts");
}

(C) Filtro por tipo y período
// costs/parser.ts (o rules)
if (!["directo","directos"].includes(String(row.tipo_gasto||row.tipoGasto).toLowerCase())) skip("not direct");

// luego del parse:

3) SQL de saneo (opcional pero recomendado; corre sólo si validás antes)

3.1 Detectar inconsistencias de período

-- Filas con mes "ago" 2025 cuyo month_key NO es 2025-08
SELECT id, cliente, proyecto, mes, "año", month_key
FROM direct_costs
WHERE "año" = 2025 AND mes ILIKE '%ago%' AND month_key <> '2025-08';


3.2 Duplicados por cliente/proyecto en agosto

SELECT cliente, proyecto, month_key, COUNT(*) cnt, SUM(costo_total) ars, SUM(monto_total_usd) usd
FROM direct_costs
WHERE month_key='2025-08'
GROUP BY 1,2,3
HAVING COUNT(*)>1
ORDER BY cnt DESC, ars DESC;


3.3 USD corrupto

SELECT id, cliente, proyecto, month_key, monto_total_usd
FROM direct_costs
WHERE month_key='2025-08' AND (monto_total_usd > 100000 OR monto_total_usd IS NULL);


Si necesitás corregir: ejecutá primero el SELECT equivalente y luego el UPDATE/DELETE dentro de una transacción. Evitá tocar el Excel; saneá sólo registros evidentemente corruptos (p.ej., month_key mal seteado o USD astronómico).
