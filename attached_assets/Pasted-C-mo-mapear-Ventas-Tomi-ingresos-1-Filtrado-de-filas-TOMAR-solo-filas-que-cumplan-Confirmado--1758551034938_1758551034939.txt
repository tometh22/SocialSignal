Cómo mapear “Ventas Tomi” (ingresos)
1) Filtrado de filas
TOMAR solo filas que cumplan:
Confirmado ∈ {“Si”, “sí”, “SI”} (case/acentos-insensitive)
Año = año del período
Mes = mes del período (mapa es→en: enero→01 … diciembre→12)
Cliente y Proyecto del Excel deben mapear al projectId activo
Nota: el filtro “Mes pasado” para hoy (sep/2025) debe resolver a agosto_2025.
2) Normalización de nombres (cliente/proyecto)
Para unir filas del Excel con tus proyectos:
normalize(s): trim → lowercase → quitar acentos → colapsar espacios
Clave canónica: key = normalize(cliente) + '::' + normalize(proyecto)
Mantener un alias map (JSON) para resolver desvíos de escritura:
{
  "warner::fee marketing": "project:warner_fee_marketing",
  "kimberly clark::fee huggies": "project:kimberly_huggies"
}
Si el par cliente/proyecto no matchea ningún alias, cae en “Proyecto sin nombre” del cliente (así evitás perder ventas, pero el objetivo es tener alias completos).
3) Moneda y monto
En cada fila:
Si Monto_USD > 0 → amountUSD = parseDec(Monto_USD)
Si Monto_USD = 0 y Monto_ARS > 0 → amountUSD = parseDec(Monto_ARS) / fx(periodo)
Si en ingresos no tenés columna de cotización, usar la FX del período que ya usás en costos (misma base ECON); si no hay FX, registrar el caso como “pendiente de cotización” y excluir de cards hasta tenerla (evita markup ficticio).
Ignorar Tipo_Venta para el cálculo del período (sirve solo para analytics); el filtro por Mes/Año ya delimita si es Fee del mes o un One Shot cargado ese mes.
parseDec robusto (ES/US, miles, moneda y paréntesis negativos):
const parseDec = (v: unknown): number => {
  if (typeof v === 'number') return v;
  const s = String(v || '').trim()
    .replace(/\s+/g, '')
    .replace(/\$/g, '')
    .replace(/\((.*)\)/, '-$1')
    .replace(/[.](?=.*\d{3}(?:[,.]|$))/g, '') // quita miles con punto
    .replace(/,(?=\d{2}$)/, '.')              // coma decimal
    .replace(/,/g, '');                       // resto comas
  const n = Number(s);
  return Number.isFinite(n) ? n : 0;
};
4) Agregación por proyecto-mes
Agrupar por projectId y sumar:
totalRealRevenueUSD(projectId, periodo) = Σ amountUSD de filas filtradas
recuentoVentas(projectId, periodo)      = # filas filtradas
5) Validaciones explícitas (evitan “números raros”)
Duplicados: misma fila (cliente+proyecto+mes+año+monto+tipo) vista más de una vez → contar una sola (usar hash de fila).
Negativos: permitirlos solo si el “Tipo_Venta” indica nota de crédito; si no, loggear y excluir.
Moneda mixta (mismo proyecto con USD y ARS el mismo mes): convertir todos a USD con la misma FX del período antes de sumar.
