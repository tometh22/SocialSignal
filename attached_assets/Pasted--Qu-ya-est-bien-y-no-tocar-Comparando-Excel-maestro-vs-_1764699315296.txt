3. Lógica correcta para CashFlow (cómo tiene que quedar)

A nivel especificación, lo que tiene que hacer el ETL es:

Fuente única: hoja CashFlow del Excel maestro.

Filtrado de filas:

Usar únicamente filas donde Ingreso/Egreso ∈ {"Ingreso", "Egreso"}.

Excluir explícitamente:

Filas con Ingreso/Egreso = "Saldo"

Filas completamente vacías.

Campos a mapear a la tabla cash_movements:

type:

"IN" si Ingreso/Egreso = "Ingreso"

"OUT" si Ingreso/Egreso = "Egreso"

bank: columna Banco

currency: columna Moneda

date: columna Fecha

monthLabel: columna Mes (por ejemplo "10 oct")

year: columna Año (por ejemplo 2025)

amountUsd: columna Monto USD (no hacer ningún otro cálculo; ya está en USD)

Agrupación por período en el ETL (syncCashFlowMovements):

Definir period = YYYY-MM usando Año + el número de mes que corresponde a Mes.

Para cada period:

cashFlowInUsd = SUM(amountUsd WHERE type = 'IN')

cashFlowOutUsd = SUM(amountUsd WHERE type = 'OUT')

cashFlowNetFromMovementsUsd = cashFlowInUsd - cashFlowOutUsd

Chequeo de consistencia:

cashFlowNetUsd sigue viniendo de Resumen Ejecutivo (columna “CashFlow”) como hasta ahora.

El warning se dispara solo si
ABS(cashFlowNetFromMovementsUsd - cashFlowNetUsd) > 1.0

Pero con el ETL correcto, para Oct-25 debería dar 0.

Con esa lógica, para Octubre 2025 la API /api/dashboard/metrics tiene que devolver:

financial.cashFlowInUsd = 112638.90

financial.cashFlowOutUsd = 45837.32

financial.cashFlowNetFromMovementsUsd = 66801.58

financial.cashFlowNetUsd = 66801.58

financial.cajaTotalUsd = -7010.68

Y el warning de inconsistencia no debería aparecer para ese mes.
