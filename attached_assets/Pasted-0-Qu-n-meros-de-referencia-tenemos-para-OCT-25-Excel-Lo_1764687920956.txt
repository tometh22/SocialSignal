2.2. Overhead operativo vs contable

Separá bien las versiones:

En monthly_financial_summary:

impuestos_usa_usd: de la función anterior (getProvisionImpuestosUsa).

En el API que arma los datos del dashboard (/api/dashboard/metrics):

// Vista financiera
financial.overheadUsd = monthly.indirect_costs_usd; // estructura
financial.provisionsUsd = monthly.provisions_usd;
financial.impuestosUsaUsd = monthly.impuestos_usa_usd; // si querés mostrarlo separado

// Vista operativa

No restes overhead ni provisiones en la vista operativa; esa carta es “después de costos de equipo”.

2.4. Conversión de moneda

Para evitar desfasajes, regla única:

En TODAS las hojas que tienen columnas tipo:

Dólares

Importe moneda de origen

Moneda país

Usá esta lógica en el parser:

function toUsd(row, fxUsdArs) {
  if (row.Dólares !== '' && row.Dólares != null) {
    return parseNumber(row.Dólares); // ya viene en USD
  }
  if (row['Moneda país'] === 'Pesos Argentina') {
    return parseNumber(row['Importe moneda de origen']) / fxUsdArs;
  }
  // si es otra moneda y tenés otro FX, ahí lo tratás;
  // si es USD explícito, devolvés el importe origen.
}


El fxUsdArs del mes lo tomás de la hoja Info Tipo de Cambio y REM para ese mismo mes (el mismo que usa Vicky para Looker).

Nada de convertir de nuevo columnas que ya están en USD.
