1. Reglas de negocio (mantener como fuente de verdad)
1.1. Vista OPERATIVO

Usar SOLO:

devengado = facturado - provision_facturacion_adelantada

directos = SUM(direct_usd)

ebit_operativo = devengado - directos

margen_operativo = ebit_operativo / devengado

markup_operativo = devengado / directos

tarifa_efectiva = devengado / horas_facturables

%facturable = horas_facturables / horas_totales

horas_totales, personas_activas, proyectos_activos

NADA de overhead, provisiones contables, impuestos ni caja en esta vista.

1.2. Vista ECONÓMICO

Usar:

devengado

directos

overhead = SUM(indirect_usd)

ebit_economico = devengado - directos - overhead

margen_economico = ebit_economico / devengado

Tampoco incluye provisiones contables, impuestos ni cash.

1.3. Vista FINANZAS

Usar:

facturado = SUM(facturacion_total)

directos

overhead

provisiones = SUM(provisions_usd)

ebit_contable = facturado - directos - overhead - provisiones

cash_in = SUM(amount_usd WHERE type = 'IN')

cash_out = SUM(amount_usd WHERE type = 'OUT')

cash_flow_neto = cash_in - cash_out

caja_total (snapshot de monthly_financial_summary)

burn_rate = directos + overhead + provisiones

runway_meses = caja_total / burn_rate

activo_total, pasivo_total, patrimonio = activo_total - pasivo_total

Esta vista sí incluye TODO lo contable + caja.

2.4. Cálculo de ALERTAS

Operativo:

X personas sin registrar horas (si lo podés calcular).

Económico:

Finanzas:

provisiones / facturado > 0.4.

pasivo_total > activo_total.

Cada alerta debe venir en un objeto:

{ type: "critical" | "warning" | "info", message: string, metric: string }
