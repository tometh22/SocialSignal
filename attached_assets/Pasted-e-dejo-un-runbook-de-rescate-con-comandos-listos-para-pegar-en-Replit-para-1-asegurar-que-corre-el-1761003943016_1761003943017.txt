4) Resolver matching cliente/proyecto y período
4.1 Obtener project_id
$PSQL <<'SQL'
SELECT prj.id, c.name AS cliente, prj.name AS proyecto
FROM dim_project prj
JOIN dim_client  c ON c.id=prj.client_id
WHERE lower(unaccent(c.name)) = lower(unaccent('${CLIENT_NAME}'))
  AND lower(unaccent(prj.name))=lower(unaccent('${PROJECT_NAME}'))
LIMIT 1;
SQL

5) Validar detalle por persona (DB vs Excel)
5.1 Detalle SoT por persona
$PSQL <<'SQL'
SELECT
  dp.name                                  AS persona,
  flm.role_name                            AS rol,
  ROUND(flm.target_hours::numeric,2)       AS horas_objetivo,
  ROUND(flm.asana_hours::numeric,2)        AS horas_asana,
  ROUND(flm.billing_hours::numeric,2)      AS horas_facturacion,
  ROUND(flm.hourly_rate_ars::numeric,2)    AS valor_hora_ars,
  ROUND(flm.cost_ars::numeric,2)           AS total_ars,
  ROUND(flm.fx::numeric,6)                 AS cotizacion,
  ROUND(flm.cost_usd::numeric,2)           AS total_usd,
  flm.flags
FROM fact_labor_month flm
JOIN dim_person  dp  ON dp.id=flm.person_id
WHERE flm.project_id=${PROJECT_ID} AND flm.period_key='${PERIOD}'
ORDER BY persona;
SQL
5.2 Totales del período (deben igualar “Suma total” del Excel)
$PSQL <<'SQL'
SELECT

  ROUND(SUM(billing_hours)::numeric,2)  AS sum_horas_fact,
  ROUND(SUM(cost_ars)::numeric,2)       AS sum_ars,
  ROUND(SUM(cost_usd)::numeric,2)       AS sum_usd
FROM fact_labor_month
WHERE project_id=${PROJECT_ID} AND period_key='${PERIOD}';
SQL

B. Render: si el RAW trae “10.270,25”, obliga UNFORMATTED_VALUE + SERIAL_NUMBER (Paso 3) y que el ETL aplique parseNum() (coma → punto).
