Objetivo general

Corregir el Dashboard Ejecutivo mensual para separar claramente:

Finanzas (FACTURADO, EBIT Contable, Burn Rate),

Operación (DEVENGADO / Ingreso Ganado, EBIT Operativo, Markup Operativo, Tarifa Efectiva),
usando la data ya existente (Rendimiento Cliente, Costos, Horas, WIP si está disponible) y que funcione para todos los meses.

1. Relevamiento rápido de la situación actual

1.1. Localizá el endpoint que alimenta el Dashboard Ejecutivo mensual (el que muestra las cards: Facturado, EBIT operativo, EBIT contable, Burn Rate, Markup, % facturables, etc.).
Probablemente está en server/routes.ts o en algún módulo tipo server/routes/dashboard.ts.
Confirmá qué ruta HTTP está usando el front (por ejemplo /api/dashboard/executive o similar).

1.2. Identificá, en ese endpoint:

De dónde sale hoy el facturado del mes (normalmente fact_rc_month con data de la pestaña “Rendimiento Cliente”).

De dónde salen los costos directos e indirectos (SoT de costos, algo como direct_costs, indirect_costs o un agregador en costs.getSummaryByPeriod()).

De dónde salen las horas trabajadas (algo tipo fact_labor_month).

Qué fórmula se usa hoy para:

EBIT operativo,

EBIT contable,

Markup,

% facturables.

Documentá mentalmente estas fórmulas, pero no cambies nada todavía.

2. Implementar “Ingreso Devengado” (Ingreso Operativo Ganado)
2.1. Concepto

Vamos a introducir un nuevo KPI: DEVENGADO (Ingreso Operativo Ganado en el período), que se basa en el trabajo efectivamente realizado y no solo en lo facturado.

Regla de negocio:

Para proyectos Fee mensuales (tipo Fee / Retainer):

El ingreso devengado del mes = facturado del mes (porque el fee corresponde exactamente al trabajo del mes).

Para proyectos One Shot / Exploratorios / Tracking con duración definida:

Se debe distribuir el valor del contrato según el avance del proyecto.

Si hay información de horas estimadas, usamos:

%_avance_acumulado = horas_reales_acum / horas_estimadas_totales, capado entre 0 y 1.

devengado_acumulado = contract_value * %_avance_acumulado.

devengado_mes = devengado_acumulado(periodo_t) - devengado_acumulado(periodo_t-1).

Si para algún proyecto no hubiera información de contrato/horas estimadas, se puede usar un fallback ya discutido:

devengado_mes ≈ costos_directos_mes * markup_base (2.5) y ajustar luego cuando haya datos mejores.

2.2. Ubicación de la lógica

2.2.1. Buscá si ya existe algo relacionado con WIP o devengado en el proyecto (palabras clave: WIP, devengado, earned, revenue_recognition, fact_devengado, etc.).

Si ya existe un módulo de devengado/WIP, reutilizalo y extendelo.

Si no existe nada, creá un módulo nuevo, por ejemplo:

// server/domain/finance/devengado.ts
export async function getDevengadoByPeriod(periodKeys: string[]): Promise<{
  totalDevengadoUsd: number;
  byProject: DevengadoProjectBreakdown[];
}> { ... }


2.2.2. Sea cual sea el enfoque, el resultado debe ser, como mínimo:

totalDevengadoUsd por período (sumado a nivel mes),

opcionalmente un breakdown por proyecto para futuro drilldown.

2.3. Data sources que el módulo puede usar

El módulo getDevengadoByPeriod puede combinar:

Ingresos / contratos:

income_sot, quotations, projects, o tablas equivalentes donde esté:

tipo de proyecto (Fee / One Shot / etc.),

valor total del contrato en USD,

fechas de inicio/fin,

horas estimadas si existen.

Horas trabajadas:

fact_labor_month o equivalente (horas por proyecto/mes).

WIP ya existente, si hay tablas con WIP acumulado.

Usá la arquitectura actual (no inventes tablas si ya existen equivalentes).
Si ves que ya hay lógica de WIP con fórmula devengado_mes = facturado_mes + ΔWIP, podés reutilizarla sin reescribir todo.

2.4. Reglas de cálculo resumidas

Implementá la lógica de forma robusta:

Para proyectos Fee → devengado_mes = facturado_mes.

Para One Shot / Exploratorios, si hay contrato + horas estimadas:

Calculá horas acumuladas vs estimadas → % avance.

Calculá devengado acumulado y devengado mensual como diferencia.

Si faltan datos para un proyecto, loggealo y usá una aproximación:

devengado_mes ≈ costos_directos_mes * markup_base (2.5)
pero marcando en logs que se está usando fallback.

El módulo debe funcionar para cualquier período que se le pase (no hardcodear 2025-10).

3. Redefinir KPIs del Dashboard Ejecutivo

Ahora modificá el endpoint del Dashboard Ejecutivo para usar el nuevo devengado.

3.1. Variables base a obtener en el endpoint

Para el período seleccionado (periodKeys):

Facturado (financiero):

Sumar revenue_usd desde fact_rc_month (Rendimiento Cliente), como ya se hace.

Devengado (operativo, nuevo):

Llamar a getDevengadoByPeriod(periodKeys) y tomar totalDevengadoUsd.

Costos directos:

Sumar costos directos en USD para el período, usando el SoT de costos (direct_costs o equivalente).

Costos indirectos:

Sumar costos indirectos en USD para el período (indirect_costs o equivalente).

Horas trabajadas:

Obtener total de horas (idealmente billable_hours, non_billable_hours y su suma) desde fact_labor_month o similar.

3.2. Nuevas fórmulas de los KPIs

Reemplazá la lógica actual de cards por:

Card FACTURADO (se mantiene igual):

Valor: facturadoUsd.

Fuente: fact_rc_month.

Card EBIT OPERATIVO (corregido):

Fórmula:
ebitOperativoUsd = devengadoUsd - directCostsUsd.

Porcentaje para mostrar como sublabel (si lo usás):
% = (ebitOperativoUsd / devengadoUsd) si devengadoUsd > 0.

Card EBIT CONTABLE:

Fórmula:
ebitContableUsd = facturadoUsd - directCostsUsd - indirectCostsUsd.

Porcentaje:
% = (ebitContableUsd / facturadoUsd) si facturadoUsd > 0.

Card BURN RATE:

Fórmula:
burnRateUsd = directCostsUsd + indirectCostsUsd.

Card MARKUP:

Fórmula:
markup = devengadoUsd / directCostsUsd
(solo si directCostsUsd > 0, sino null o 0).

Tarifa Efectiva (si decidís mostrarla en alguna card o tooltip):

Fórmula:
tarifaEfectiva = devengadoUsd / totalHours
(solo si totalHours > 0).

3.3. Mantener KPIs financieros que ya existen

Si el dashboard muestra cosas como:

Cash Flow del mes,

Balance,

% de margen contable,

dejá la lógica actual (basada en fact_rc_month + costos totales) sin tocar, porque eso es lo que usa administración para el Excel.

La diferencia es:

Capa financiera → usa facturado.

Capa operativa → usa devengado.
