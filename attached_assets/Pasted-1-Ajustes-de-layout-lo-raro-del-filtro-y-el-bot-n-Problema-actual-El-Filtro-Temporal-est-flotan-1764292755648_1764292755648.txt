4) Checklist técnico backend + front

Usalo para revisar rápido que todo esté bien armado.

4.1. Backend – Datos y lógica

Contrato de respuesta

El endpoint del Executive Dashboard devuelve:

financial y operational como bloques separados.

Nombres exactamente iguales a los que usa el front.

No se devuelven null donde el front espera número (usar 0).

Período

El filtro envía periodKey tipo YYYY-MM.

El backend usa SOLO ese período para agrupar y sumar.

No hay hardcode de 2025-10 en ningún lado.

Fuentes

financial.billedUsd sale de fact_rc_month (pestaña Rendimiento Cliente).

directCostsUsd e indirectCostsUsd salen de Costos SoT (no de tablas legacy).

earnedUsd (devengado) sale del SoT de devengado (o de la lógica que definimos, pero no se iguala ciegamente a facturado).

Fórmulas

ebitAccountingUsd = billedUsd - (direct + indirect).

ebitOperationalUsd = earnedUsd - directCosts.

markup = directCosts > 0 ? earnedUsd / directCosts : 0.

effectiveRate = billableHours > 0 ? earnedUsd / billableHours : 0.

billableRatio = totalHours > 0 ? billableHours / totalHours : 0.

Sanity checks con Excel

Para un mes con datos (ej. Octubre):

Facturado back = facturado Excel.

Costos directos/indirectos back = pestaña Costos Directos e Indirectos.

Horas totales = dashboard legacy de horas (Asana).

Para un mes sin datos: todos los KPIs = 0, sin errores.

Manejo de errores

Si el query a DB falla → responde 500 con mensaje claro.

El front muestra un estado “Error al cargar datos” en lugar de 0 falsos.

4.2. Frontend – Conexión y render

Consumo del endpoint

Hook o service centralizado, p.ej. useExecutiveDashboard(periodKey).

Maneja loading / error / success.

Mapping de datos

Vista Operativa usa solo campos de operational en cálculos.

Vista Financiera usa solo campos de financial.

Ningún cálculo de UI usa directamente billedUsd para métricas operativas.

Formateo

Números en formato LATAM:

Moneda: $42.100 / $64.2k según el diseño que quieras, pero consistente.

Porcentajes: siempre con %, 1 decimales max (58.7%).

Markup: 2.42x.

Redondeo centralizado (helper de formato, no repetido en cada card).

Filtro Temporal

Cada cambio de mes dispara:

actualización del estado periodKey.

nueva llamada al endpoint.

El estado visual del mes seleccionado siempre coincide con el período que se está usando en el request.

Toggle Operativo/Financiero

Cambia solo lo que se muestra.

No dispara nuevas llamadas al backend (los datos vienen juntos en una sola respuesta).
