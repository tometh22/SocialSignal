Arreglo estructural (recomendado)
A) Normalizá el estado en “slices” (evitá unifiedData monolítico)
type ProjectState = {
  summary?: Summary;
  quotation?: Quotation;
  actuals?: Actuals;
  metrics?: Metrics;
  timeEntries?: TimeEntry[];
  currencyNative?: 'ARS'|'USD';
  // opcional: version por slice
  _v: { summary: number; quotation: number; actuals: number; metrics: number; timeEntries: number; };
};
Beneficio: cada fetch actualiza solo su slice y no borra el resto.
B) Usa useReducer con acciones por slice + versionado anti-carrera
type Action =
  | { type: 'SUMMARY_OK'; payload: Summary; reqId: number }
  | { type: 'ACTUALS_OK'; payload: Actuals; reqId: number }
  // ...

function reducer(state: ProjectState, action: Action): ProjectState {
  switch (action.type) {
    case 'SUMMARY_OK':
      if (action.reqId < state._v.summary) return state; // descarta respuesta vieja
      return {
        ...state,
        summary: action.payload,
        _v: { ...state._v, summary: action.reqId }
      };
    // ...
  }
}
Generá un reqId incremental por llamada o timestamp; descartá respuestas atrasadas.
C) AbortController por fetch
Si el usuario cambia de proyecto o se re-renderiza, abortá requests anteriores para que no sobreescriban:
const ac = new AbortController();
fetch(url, { signal: ac.signal }).then(...);
return () => ac.abort();
D) Selectores con fallback “inteligente”
Centralizá la lógica de presentación:
const selectors = {
  costDisplay(state: ProjectState) {
    return state.summary?.costDisplay
        ?? state.actuals?.totalWorkedCost
        ?? 0;
  },
  currency(state: ProjectState) {
    return state.summary?.currencyNative ?? state.currencyNative ?? 'ARS';
  },
};
En el JSX, usá selectors.costDisplay(state) y no el objeto crudo.
E) useMemo sólo sobre claves estables
No dependas del objeto; dependé de propiedades:
const perf = useMemo(() => {
  const used = selectors.budgetUtilization(state);
  const eff  = selectors.efficiency(state);
  return { used, eff };
}, [state.metrics?.budgetUtilization, state.metrics?.efficiency]);

Snippet de reducer + merge seguro
import { merge } from 'ts-deepmerge'; // o implementa uno propio sin mutar arrays si no querés

function deepMerge<T>(a: T, b: Partial<T>): T {
  return merge.withOptions({ mergeArrays: false }, a, b);
}

const initial: ProjectState = { _v: { summary:0, quotation:0, actuals:0, metrics:0, timeEntries:0 } };

function useProjectState(projectId: string) {
  const [state, dispatch] = useReducer(reducer, initial);

  useEffect(() => {
    let req = Date.now();
    const ac = new AbortController();
    fetch(`/api/projects/${projectId}/summary`, { signal: ac.signal })
      .then(r => r.json())
      .then(data => SummarySchema.parse(data))
      .then(summary => dispatch({ type:'SUMMARY_OK', payload: summary, reqId: req }))
      .catch(()=>{ /* handle/ignore abort */ });
    return () => ac.abort();
  }, [projectId]);

  return state;
}
