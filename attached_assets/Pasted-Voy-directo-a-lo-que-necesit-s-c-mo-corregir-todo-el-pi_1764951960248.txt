0. Contexto que el dev tiene que respetar

Las métricas de la app deben calcar lo que hoy sale en:

pestaña Resumen Ejecutivo – Desglose

La app tiene dos vistas:

2. Reglas de negocio para cada métrica

Ahora el agent tiene que codificar las fórmulas únicas que la app va a usar.

2.1. Definiciones base

Para cualquier month_key:

facturado = monthly_summary.ventas_mes_usd

directos_reales = monthly_costs.direct_cost_usd

indirectos_reales = monthly_costs.indirect_cost_usd

provisiones_totales = monthly_provisions.provisions_total_usd

prov_facturacion_adelantada = monthly_summary.pasivo_facturacion_adelantada
(si no existe para ese mes, usar 0).

2.2. Ingreso Devengado (operativo)

Regla simple y estable:

devengado = facturado - prov_facturacion_adelantada


Esta diferencia representa trabajo aún no realizado (facturación anticipada).

2.3. Separar costos operativos vs contables

Para OPERATIVO:

costos_directos_operativos = directos_reales
(sin provisiones; si en la tabla de costos hay filas marcadas como provisión, excluir).

overhead_operativo = indirectos_reales
(también sin provisiones).

Para FINANCIERO:

costos_directos_financieros = directos_reales

overhead_financiero = indirectos_reales

provisiones_financieras = provisiones_totales
(no las metas dos veces; si ya vinieron como filas en costos, asegurate de no duplicarlas).

2.4. Métricas OPERATIVAS (vista Operativo)

Para el endpoint operativo:

devengado (como en 2.2)

ebit_operativo = devengado - costos_directos_operativos

margen_operativo_real = ebit_operativo / devengado (si devengado > 0)

markup_operativo = devengado / costos_directos_operativos (si directos > 0)

tarifa_efectiva = devengado / hours_facturables (si hours_facturables > 0)

%_horas_facturables = hours_facturables / hours_total

costos_operativos_totales = costos_directos_operativos + overhead_operativo

Estos son los valores que deben alimentar:

Card “Ingreso Devengado”

Card “EBIT Operativo”

Card “Tarifa efectiva”

Card “Markup operativo”

Card “% Horas facturables”

Barra “Composición de costos operativos”

2.5. Métricas FINANCIERAS (vista Financiero)

Para el endpoint financiero:

facturado

costos_directos_financieros

overhead_financiero

provisiones_financieras

burn_rate = costos_directos_financieros + overhead_financiero + provisiones_financieras

ebit_contable = facturado - burn_rate

cashflow = monthly_summary.cashflow_excel

caja_total = monthly_summary.activo_liquido (o la métrica que uses para Caja).

Cards:

“Facturado”

“EBIT contable (incluye provisiones)”

“Directos”

“Overhead”

“Provisiones”

“Burn Rate”

“Cash Flow Neto”

“Caja total”

3. Backend: endpoints y filtro temporal

Instrucciones directas para Replit Agent:

En el backend, localizá el endpoint que devuelve los datos del “Dashboard Ejecutivo” (por ejemplo /api/dashboard/executive o similar). Vamos a unificar la lógica.

3.1. Parámetros del endpoint

El endpoint debe recibir:

month en formato YYYY-MM

view en { "operativo", "financiero" }

Ejemplo:
GET /api/dashboard/executive?month=2025-09&view=operativo

3.2. Lógica interna del endpoint

Pseudocódigo que el agent debe traducir a tu stack:

const { month, view } = req.query;

// 1) Leer datos base
const summary = await monthlySummaryRepo.getByMonth(month);
const costs   = await monthlyCostsRepo.getByMonth(month);
const prov    = await monthlyProvisionsRepo.getByMonth(month);
const hours   = await monthlyHoursRepo.getByMonth(month);

// 2) Variables comunes
const facturado = summary.ventas_mes_usd || 0;
const directos  = costs.direct_cost_usd || 0;
const indirectos = costs.indirect_cost_usd || 0;
const provFactAdelantada = summary.pasivo_facturacion_adelantada || 0;
const provisiones = prov.provisions_total_usd || 0;

const devengado = facturado - provFactAdelantada;

if (view === 'operativo') {
  const horasFact = hours.hours_facturables || 0;
  const horasNoFact = hours.hours_no_facturables || 0;
  const horasTotal = horasFact + horasNoFact;

  const ebitOper = devengado - directos;
  const margenOperReal = devengado > 0 ? ebitOper / devengado : 0;
  const markupOper = directos > 0 ? devengado / directos : 0;
  const tarifaEfectiva = horasFact > 0 ? devengado / horasFact : 0;
  const pctHorasFact = horasTotal > 0 ? horasFact / horasTotal : 0;
  const totalOper = directos + indirectos;

  return res.json({
    view: 'operativo',
    month,
    facturado,
    devengado,
    ebit_operativo: ebitOper,
    margen_operativo_real: margenOperReal,
    markup_operativo: markupOper,
    tarifa_efectiva: tarifaEfectiva,
    pct_horas_facturables: pctHorasFact,
    horas_facturables: horasFact,
    horas_no_facturables: horasNoFact,
    horas_totales: horasTotal,
    costos_directos: directos,
    overhead_operativo: indirectos,
    costos_operativos_totales: totalOper,
  });
}

if (view === 'financiero') {
  const burnRate = directos + indirectos + provisiones;
  const ebitCont = facturado - burnRate;

  return res.json({
    view: 'financiero',
    month,
    facturado,
    directos: directos,
    overhead: indirectos,
    provisiones,
    burn_rate: burnRate,
    ebit_contable: ebitCont,
    cashflow: summary.cashflow_excel || 0,
    caja_total: summary.activo_liquido || 0,
    balance_neto: summary.balance_neto || 0,
  });
}


El agent debe adaptar esto al lenguaje (TS/JS/Python) y estructura real del proyecto.
