Nueva clasificación de costos

Necesitamos introducir una separación explícita de costos en el ETL.

Para cada fila que viene de las hojas de costos, clasificá así:

cost_type: "direct" | "indirect"

direct = costos de equipo imputados a proyectos (lo que hoy aparece como “Directos (equipo)”).

indirect = todo lo demás (overhead, provisiones, impuestos, etc.).

is_provision: boolean

true si la fila corresponde a provisiones o ajustes contables.

false si es overhead operativo real.

provision_kind: string opcional

"cliente_pepsico", "cliente_warner", "impuestos_usa", "impuestos_iva", "otra" según corresponda.

Criterio para marcar is_provision = true:

Si en cualquier columna de texto (subtipo, descripción, cuenta, proyecto) aparece alguno de estos patrones (case insensitive):

"PROVISION"

"PROVISIÓN"

"PEPSICO"

"WARNER"

"IMPUESTOS USA"

"IMPUESTO USA"

"TAX USA"

O si la fila proviene directamente de las pestañas:

Provisión pasivo proyectos

Impuestos

cost_type = "indirect"

is_provision = true

provision_kind según texto detectado.
