3) Invariante Labor vs RC (Δ ≤ 0.05 USD por redondeo)
# Cambiá el project_id si querés (34 o 39)
for PID in 34 39; do
  echo "== Proyecto $PID =="
  $PSQL <<SQL
SELECT period_key,
       ROUND(SUM(cost_usd)::numeric,2)                    AS labor_cost_usd,
       (SELECT ROUND(cost_usd::numeric,2) FROM fact_rc_month r
         WHERE r.project_id = $PID AND r.period_key = flm.period_key) AS rc_cost_usd,
       ROUND(SUM(cost_usd)::numeric,2) -
       (SELECT ROUND(cost_usd::numeric,2) FROM fact_rc_month r
         WHERE r.project_id = $PID AND r.period_key = flm.period_key) AS delta
FROM fact_labor_month flm
WHERE project_id = $PID
GROUP BY period_key
ORDER BY period_key;
SQL
done
Pasa si: todas las filas muestran |delta| ≤ 0.05.
(Warner agosto puede dar ~42 si la planilla RC difiere de la de costos; eso es una discrepancia “aceptable si documentada”).
4) Consistencia por persona (muestra mensual)
Verifica que cost_ars ≈ billing_hours × hourly_rate_ars.
# Muestra: Warner agosto 2025
$PSQL <<'SQL'
SELECT person_key,
       ROUND(billing_hours::numeric,2)     AS hours_billing,
       ROUND(hourly_rate_ars::numeric,2)   AS rate_ars,
       ROUND(cost_ars::numeric,2)          AS cost_ars,
       ROUND((billing_hours*hourly_rate_ars)::numeric,2) AS calc_ars,
       ROUND(cost_ars - (billing_hours*hourly_rate_ars),2) AS diff
FROM fact_labor_month
WHERE project_id = 34 AND period_key = '2025-08'
ORDER BY cost_ars DESC
LIMIT 10;
SQL
Pasa si: diff ∈ [-0.1, 0.1] (redondeo).
