4. SEPARACIÓN DE LÓGICA (LO MÁS IMPORTANTE)

Hoy “financiero” mezcla devengado, contabilidad, overhead y caja.
Eso es lo que está generando contradicciones.

Debés dividir:

4.1 ENDPOINT OPERATIVO

Ruta: /api/executive/operativo

Tablas permitidas:

monthly_financial_summary (solo para facturado & provisión)

fact_cost_month (solo directos)

fact_labor_month (horas)

Fórmulas:

devengado = facturado - provision_facturacion_adelantada

directos = SUM(direct_usd)

ebit_operativo = devengado - directos

markup = devengado / directos

tarifa_efectiva = devengado / horas_facturables

%facturable = horas_facturables / horas_totales

No mostrar aquí:

overhead

provisiones contables

facturado como métrica

burn rate

caja

impuestos

pasivos

cashflow

4.2 ENDPOINT ECONÓMICO

Ruta: /api/executive/economico

Tablas permitidas:

monthly_financial_summary

fact_cost_month (directos + overhead)

fact_rc_month (performance por cliente/proyecto)

Fórmulas:

devengado = facturado - provision_adelantada

directos = SUM(direct_usd)

overhead = SUM(indirect_usd)

ebit_economico = devengado - directos - overhead

margen_economico = ebit_economico / devengado

Por cliente/proyecto: (desde fact_rc_month)

revenue_usd

cost_usd

profit

markup

margin

No mostrar aquí:

provisiones contables

impuestos USA

caja

cashflow

burn rate

facturado

4.3 ENDPOINT FINANZAS (Contable + Caja)

Ruta: /api/executive/finanzas

Tablas permitidas:

monthly_financial_summary

pl_adjustments (provisiones, impuestos)

cash_movements

Activo, Pasivo (según tu estructura)

Fórmulas:

facturado = SUM(facturacion_total)

directos = SUM(direct_usd)

overhead = SUM(indirect_usd)

provisiones = SUM(provisions_usd)

total_contable = directos + overhead + provisiones

ebit_contable = facturado - total_contable

beneficio_neto = ebit_contable - ajustes_pnl

cash_in = SUM(amount_usd WHERE IN)

cash_out = SUM(amount_usd WHERE OUT)

cashflow_neto = cash_in - cash_out

caja_total = snapshot monthly_financial_summary

burn_rate = total_contable

runway = caja_total / burn_rate

activo_total, pasivo_total, patrimonio

No mostrar aquí:

devengado

ebit_operativo

markup operativo

horas
