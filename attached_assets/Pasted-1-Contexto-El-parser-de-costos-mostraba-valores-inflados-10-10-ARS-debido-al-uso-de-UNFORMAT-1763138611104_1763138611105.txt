1. Contexto

El parser de costos mostraba valores inflados (10¹⁴–10¹⁶ ARS) debido al uso de UNFORMATTED_VALUE en la API de Google Sheets.
Además, se habían detectado registros rechazados por filtros erróneos en tipoCosto y pérdidas de campo cliente.

2. Cambios realizados

Archivos modificados

server/services/googleSheetsWorking.ts

server/domain/costs/parser.ts

Ajustes principales

Sustituido valueRenderOption: 'UNFORMATTED_VALUE' → 'FORMATTED_VALUE' en las cuatro funciones:

getCostosDirectosIndirectos()

getProyectosConfirmados()

getVentasTomi()

getRendimientoCliente()

Consolidado parser monetario con parseMoneySmart() para soportar formatos “$1,234.56”, “1.234,56”, “ARS 1.200,00”.

Integrado helper normalizeCostRow() para estandarizar campos (tipoCosto, categoria, cliente).

Rehabilitado requireAuth en /api/costs.

Eliminado logging temporal (console.log, DEBUG Sep-2025, etc.).

Añadido fallback categoria → tipoCosto para datos legacy.
