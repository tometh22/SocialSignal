Quiero que implementes, end-to-end, la ingesta del Excel maestro “Seguimiento Financiero_Economico Para Tablero” (hoja “Resumen Ejecutivo”) a una tabla de BD de resumen mensual, y que el dashboard de la app use ESO como fuente para los KPIs financieros tipo informe de Administración.

0. Contexto

El Excel ya existe y se usa para otras cosas.

Administración me manda un informe mensual con estos KPIs (ejemplo Octubre):

Balance neto ≈ 68.210

EBIT operativo ≈ 42.300

Beneficio Neto ≈ 21.666 (incluye provisiones, ej. Pepsico)

Cash Flow neto ≈ 66.802

Markup promedio (hoy no nos importa el 4,75 exacto, eso puede seguir en el Excel).

Mi app debe poder mostrar estos KPIs para cualquier mes, no sólo octubre.

Todo lo que hagas debe ser genérico por mes (period_key YYYY-MM), usando la hoja “Resumen Ejecutivo” como single source of truth para KPIs financieros mensuales.

1. Crear tabla monthly_financial_summary en el schema

Abre shared/schema.ts.

Crea una nueva tabla Drizzle monthly_financial_summary con estas columnas (usa tipos razonables; numéricos como numeric/double precision, fechas como date, strings como text):

id (serial, PK)

period_key (text, YYYY-MM, UNIQUE)

year (integer)

month_number (integer, 1–12)

month_label (text, por ejemplo “10 oct” tal como viene en el Excel)

cierre_date (date; la fecha “Cierre” si está disponible en la hoja, si no puede ser null)

KPI financieros clave (todos numéricos):

total_activo // TOTAL ACTIVO

total_pasivo // TOTAL PASIVO

balance_neto // BALANCE NETO (Activo – Pasivo)

caja_total // CAJA TOTAL

inversiones // Inversiones / Crypto (activo mediano plazo / inversiones)

Cashflow:

cashflow_ingresos // Ingresos reales del mes

cashflow_egresos // Egresos reales del mes

cashflow_neto // Cash Flow neto del mes (coincide con “Cash Flow: 66.802”)

Cuentas:

cuentas_cobrar_usd // Cuentas a cobrar (USD)

cuentas_pagar_usd // Cuentas a pagar (USD)

Facturación y costos (visión contable/financiera):

facturacion_total // Ventas del mes facturadas (ARS+USD)

costos_directos // Costos directos contables del mes

costos_indirectos // Costos indirectos contables del mes

iva_compras // IVA compras

impuestos_usa // Impuestos USA (del mes)

Resultados:

ebit_operativo // EBIT (Utilidad operativa)

beneficio_neto // Beneficio neto (ya con provisiones)

markup_promedio // Markup promedio global del mes (el que hoy está en el Resumen)

Metadatos:

created_at (timestamp con default now())

updated_at (timestamp con default now(), on update)

Ejecuta la migración o db:push para crear la tabla en la BD.

3. ETL: poblar monthly_financial_summary desde “Resumen Ejecutivo”

Abre server/etl/sot-etl.ts (o el ETL principal que ya procesa el Excel maestro).

Agrega un nuevo paso en el pipeline, por ejemplo syncResumenEjecutivoToMonthlyFinancialSummary().

Implementa la función así:

Llama a getResumenEjecutivo() para traer todas las filas.

Para cada fila:

Construye month_number a partir de mesLabel y/o de la fecha de cierre:

Ejemplo robusto:

Si tienes cierre con una fecha → usa esa fecha:

year = cierre.getFullYear()

month_number = cierre.getMonth() + 1

Si no, parsea mesLabel ("10 oct") + year.

Construye period_key como YYYY-MM (mes con 2 dígitos).

Mapea todos los campos numéricos a las columnas de monthly_financial_summary con conversión segura (NaN → null).

Implementa un upsert por period_key:

Si ya existe monthly_financial_summary.period_key = X → UPDATE.

Si no existe → INSERT.

Deja created_at/updated_at correctos.

Integra esta función al flujo del ETL general:

En el lugar donde ya sincronizas otras tablas (fact_rc_month, fact_cost_month, etc.), añade:

await syncResumenEjecutivoToMonthlyFinancialSummary();

Ejecuta el ETL una vez (por consola o vía endpoint ya existente) y verifica en la BD que tengas registros para todos los meses históricos (SELECT * FROM monthly_financial_summary LIMIT 20).

Abre server/routes.ts y localiza el endpoint del Executive Dashboard (algo como /api/dashboard/executive-summary o equivalente).

Actualmente el facturado y algunos KPIs los toma de fact_rc_month, CashFlow derivado, etc.

Quiero esto:

Mantener la parte OPERATIVA (devengado, costos directos operativos, horas, etc.) tal como la tenemos.

Cambiar la parte FINANCIERA (Balance, EBIT contable, Beneficio Neto, Cashflow, Ctas a cobrar/pagar, Facturación contable) para que lea de monthly_financial_summary.

Dentro del handler:

A partir del período solicitado (period_key o period query YYYY-MM que ya usas), haz algo así:

const [summary] = await db.select().from(monthlyFinancialSummary)
  .where(eq(monthlyFinancialSummary.periodKey, periodKey));

if (!summary) {
  // fallback: devolver nulls o 0 para la capa financiera, pero NO romper el endpoint
